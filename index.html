How It Works

From instruction
to action

WebSense breaks every task into three clean phases: capture, reason, act.

01
Capture the page
WebSense snapshots the full DOM — every interactive element gets a stable ID and numbered overlay so the AI knows exactly what it can click, type, or navigate to.
02
Think & Plan
Your task and the page snapshot are sent to the selected AI model. It reasons step-by-step and produces a structured action plan — visible live in the conversation feed.
03
Execute actions
The backend translates the AI's plan into real browser actions via Chrome scripting — clicks, form fills, navigation — up to 12 steps per task, with a pause between each.
04
Review & Improve
After each task, rate the outcome. Successful trajectories are saved as memory; failed ones become lessons that make future runs smarter.

OpenRouter — One key, all models

A single OpenRouter API key gives you access to GPT, Gemini, Claude, Llama, Qwen and dozens more. Switch models without ever touching your settings again.

Fully local with Ollama

Point WebSense at your local Ollama server. No API key, no data leaving your machine. Pull any model with ollama pull and it appears instantly in the dropdown.

llama.cpp server support

Connect to a running llama.cpp server for even more control over quantization, context length, and hardware acceleration.

Get Started

Up and running
in 4 steps

No build step, no account required. Just clone, install, and load into Chrome.

1
Start the backend
Clone the repo and run the FastAPI backend server.
uvicorn main:app --reload
2
Open Extensions
In Chrome, navigate to the extensions page and enable Developer Mode.
chrome://extensions/
3
Load Unpacked
Click "Load unpacked" and select the extension/ directory from the repo.
Load unpacked → /extension
4
Pin & Go
Pin WebSense to your toolbar, click it to open the side panel, add your API key and start your first task.

Research & Fine-Tuning

On-device intelligence,
